║              📋 HOW TO START NGROK FROM YOUR TERMINAL                ║
║                                                                      ║
╚══════════════════════════════════════════════════════════════════════╝

🚀 QUICK START:

Open a new terminal and run:

    ngrok http 8888

That's it! Keep the terminal open.

══════════════════════════════════════════════════════════════════════

📖 DETAILED STEPS:

1. Open a NEW terminal window (⌘T or ⌘N)

2. Type this command:
   
   ngrok http 8888

3. Press Enter

4. You'll see the ngrok dashboard with your public URL

5. Look for the "Forwarding" line - that's your public URL!
   Example: https://zestfully-chalky-nikia.ngrok-free.dev

══════════════════════════════════════════════════════════════════════

✅ IF NGROK IS ALREADY RUNNING:

If you see an error like "endpoint already online":

1. Find the terminal where ngrok is running
2. Press Ctrl+C to stop it
3. Then run again: ngrok http 8888

• Port 8888 must be free (reverse proxy uses it)

🔧 TROUBLESHOOTING:

Problem: "ngrok: command not found"
Solution: Install ngrok:
   brew install ngrok

Problem: "endpoint already online"
Solution: Stop existing ngrok first (Ctrl+C), then start again

Problem: ngrok connects but URLs return 404
Solution: Make sure the reverse proxy is running on port 8888:
   cd /Users/dgolani/Documents/AI_Studio/backend
   python3 reverse_proxy.py

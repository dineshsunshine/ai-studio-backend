# Video Generation Mock Mode - Frontend Integration Prompt

**IMPORTANT:** `mockMode` is now **REQUIRED** in all video generation API calls.

When creating a video job via `POST /api/v1/video-jobs`, you **MUST** include `mockMode` in your FormData:

```javascript
formData.append('mockMode', 'true');   // REQUIRED: 'true' to skip Veo API, 'false' to use Veo API
// ... other fields ...
```

**Behavior:**
- `mockMode: 'true'` → Skips Google Veo API, returns test video after 8-15 seconds
- `mockMode: 'false'` → Normal video generation via Google Veo API
- **Missing `mockMode`** → API will return `400 Bad Request` error

**Details:**
- Mock mode still consumes 50 tokens (simulates real behavior)
- Random 8-15 second delay simulates real generation time
- Returns test video URL from Cloudinary
- Same response format as real mode

**Example:**
```javascript
const formData = new FormData();
formData.append('prompt', 'Your prompt');
formData.append('model', 'veo-3.1-fast-generate-preview');
formData.append('resolution', '720p');
formData.append('aspectRatio', '16:9');
formData.append('durationSeconds', 4);
formData.append('mockMode', 'true');  // <--- ADD THIS!
// ... rest of your code
```

Use mock mode for UX/UI testing to avoid API costs during development.
